# Results Directory

## Canonical numbers

**`results_table.csv` / `results_table.json`** are the single source of truth for all experiment results.
Generated by `scripts/results_table.py` — re-run that script to refresh after new experiments land.

The canonical headline numbers for the paper are drawn from two categories:

| Category | `kind` column | Datasets | What it covers |
|---|---|---|---|
| **Baseline (memmap)** | `training` | `*_memmap` / `*_qwen3_memmap` | All trained probes and our method, re-run on the memmap data pipeline (6 datasets × 2 models × 5 seeds) |
| **Sampling** | `sampling` | all except MMLU | SE and SelfCheckGPT baselines (no seeds — deterministic given the sampled generations) |

When assembling paper tables or figures, filter to these two categories only. The zarr-dataset rows (`kind=training`, dataset name without `_memmap` suffix) are **not** canonical main-paper numbers — see below.

## §7.1 loss-decomposition ablation

**`loss_decomposition_table.csv` / `loss_decomposition_table.json`** are the canonical files for the 2×2 ablation that attributes performance to the two loss components. Auto-generated by `scripts/results_table.py` alongside the main table — same schema, filtered to the four methods below.

|                         | **− logprob-recon**    | **+ logprob-recon**         |
|---                      |---                     |---                          |
| **SAPLMA head**         | `saplma`               | `saplma_logprob_recon`      |
| **Contrastive head**    | `contrastive`          | `contrastive_logprob_recon` |

Coverage notes:

- `saplma`, `saplma_logprob_recon`, `contrastive_logprob_recon` — full memmap coverage (6 datasets × 2 models × 5 seeds).
- `contrastive` (no recon) — zarr only; hotpotqa + popqa at 5 seeds × 2 models. The ablation uses these zarr results (no memmap runs planned).

These rows are duplicates of cells already present in `results_table.{json,csv}` (still `kind="training"`); the separate file just makes the ablation easy to load without scanning the full table. They remain in the main table as well — `contrastive_logprob_recon` is the headline method for §5 and is not an ablation-only row.

## §6 cross-dataset transfer matrix

**`transfer_matrix_table.csv` / `transfer_matrix_table.json`** are the canonical files for the §6 cross-dataset transfer view (issues #89 / #113). Auto-emitted by `scripts/results_table.py` from per-cell JSONs under `runs/transfer_matrix_memmap/{llama,qwen3}/` (written by `scripts/eval_transfer_matrix_memmap.py`).

Schema differs from the main table: each row carries `kind="transfer"` with both `source_dataset` and `target_dataset` columns — one cell per (model, method, source, target, seed, metric). Pass `--no-transfer` to `results_table.py` to skip emission.

## Per-sample predictions (`preds/`)

The `preds/` cache holds per-sample prediction files used by `scripts/predictions_gap_report.py` and for downstream analysis (calibration, ensembling, error inspection). Headline AUROC numbers do **not** depend on it — they live in `results_table.csv`.

It is gitignored. To populate it, either:

- **Download the release tarball** (296 MB compressed, 838 MB extracted, 712 files):
  ```bash
  mkdir -p results/preds
  curl -L -o /tmp/preds.tar.gz \
    https://github.com/hyang0129/HalluLens/releases/download/preds-bundle-2026-05-21/predictions_bundle.tar.gz
  tar -xzf /tmp/preds.tar.gz -C results/preds/
  ```
- **Rebuild from cluster runs**: dispatch `scripts/assemble_predictions_tarball.py` on a GPU node (heavy `find`/`tar` over `runs/baseline_comparison_*_memmap/`), then `scp` the resulting `shared/predictions_bundle.tar.gz` down and extract.
