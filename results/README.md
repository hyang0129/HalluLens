# Results Directory

## Canonical numbers

**`results_table.csv` / `results_table.json`** are the single source of truth for all experiment results.
Generated by `scripts/results_table.py` — re-run that script to refresh after new experiments land.

The canonical headline numbers for the paper are drawn from two categories:

| Category | `kind` column | Datasets | What it covers |
|---|---|---|---|
| **Baseline (memmap)** | `training` | `*_memmap` / `*_qwen3_memmap` | All trained probes and our method, re-run on the memmap data pipeline (6 datasets × 2 models × 5 seeds) |
| **Sampling** | `sampling` | all except MMLU | SE and SelfCheckGPT baselines (no seeds — deterministic given the sampled generations) |

When assembling paper tables or figures, filter to these two categories only. The zarr-dataset rows (`kind=training`, dataset name without `_memmap` suffix) are **not** canonical main-paper numbers — see below.

## Zarr results (ablation use only)

Two methods have results only in zarr-format datasets (hotpotqa + popqa, 5 seeds):

| Method | Purpose |
|---|---|
| `contrastive` | Contrastive loss alone, no logprob-recon auxiliary |
| `saplma_logprob_recon` | SAPLMA architecture + logprob-recon auxiliary |

These exist specifically to support the **§7.1 loss decomposition ablation**: comparing contrastive-only vs. logprob-recon-only (`saplma_logprob_recon`) vs. the full model (`contrastive_logprob_recon`, which has full memmap coverage) lets us attribute how much each loss component contributes. They are not used in the §5 main results table.
